Technical Blog - Problem Solving

By Sam Manongga | Post Date: March 29 2017

Tell your non-tech friend a story about a time you got blocked on a simple problem and solved it

In my previous job working at a school in Beijing, China, I had to create program that calculate the Air Quality Indexof the area. At that time Beijing had a serious problem with air pollution and the school administrator want to be able to let the teacher know if it was safe to send the students out during break time. They had a machine that spewed out raw data every 15 minutes and those need to be calculated and stored in database.

When I first got the assignment I was a bit nervous because I don’t know where to begin because there are just so many if and conditions before you can plot them to AQI formula. Doing pseudo coding helped me to map out the structure and be able to spot which code should be grouped together.

When working on the calculation part, I decided to go try out the formula as if I already had the raw data mapped out. It worked pretty well during soft launch. However, one day it gave an incorrect answer where it should be “Heavily Polluted”, it gave out “Lightly Polluted”.

Ok. The thing is that there was no error messages in the console.logging. Previous data were correct. I thought it had to be the condition statement… nope. I tried googling but didn’t see anyone with the same problem or did something similar to what I was doing.

Since I was the only web developer in the department, I didn’t have anyone to ask for guidance. So, I ended up skyping my brother who is a software engineer. I walked him through my program and then all sudden I got it… Should been easy to solve but I didn’t see it…. I was too focus on thinking that it was a big one but actually I had the mathematical order of operation wrong. LoLz.

Problem solving techniques

  • Pseudocode
     This technique is the first I would use when I do programming. I think this is similar to when wire framing a website.
  • Trying something 
    I would use this after as it is to test out the pseudocode.
  • Rubber ducky method 
    I haven’t tried this one out.
  • Reading error messages
     Method that I have started using. If I don’t know the message then I would search what the error message means but most of the time, its pretty straight forward.
  • Console.logging 
    I think this one is useful too and I’ve done it while working with DOM manipulation task to see if it output the correct CSS class
  • Googling 
    Definitely a to go method to problem solving. This will be the last resort before I go ask a person.
  • Asking your peers for help
     I haven’t tried this one out but I think I would definitely be doing it.
  • Asking coaches for help
     Same.
  • Improving your process with reflection
     I have done it every week as part of the assignment not sure if I would do it unless this become a habit.

Explain to your non-tech friend

  • .map()
     It takes array and function as arguments. Then applies the function to every content in the array. When successful, it will returns the new array content.
  • .filter() 
     A method to remove unwanted content in array based on a condition
  • .reduce()
      A function to combine each content of an array from left to right
